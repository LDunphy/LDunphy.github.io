3. Designing for User Experience (UX)

The best visualisations are designed with the user in mind. A good user experience means the chart is not only accurate and beautiful, but also intuitive, relevant, and easy for the intended audience to understand. Clarity trumps complexity.

UX Principles: Do's and Don'ts

‚úÖ DO: Use Color with Purpose

Use a limited, accessible color palette to highlight key data or group related items. Color should add meaning, not just decoration.

‚ùå DON'T: Create a "Rainbow"

Using too many colors without clear logic creates visual clutter ("chartjunk") and confuses the viewer, hiding the key message.

‚úÖ DO: Keep It Simple

Remove any non-essential elements. Every gridline, label, and axis should serve to clarify the data. If it doesn't, remove it.

‚ùå DON'T: Use the Wrong Chart

Using a pie chart for non-compositional data or a line chart for discrete categories is fundamentally misleading. Match the chart to the data's story.
